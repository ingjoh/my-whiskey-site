# Proposal

A **Proposal** is a sales quote generated by an admin or operator. It outlines pricing options, duration, and vessel availability for a guest's prospective excursion, which can be converted into a Booking upon guest deposit completion.

## Key Attributes
*   Contains custom pricing adjustments or package rates.
*   Binds a guest's contact info temporarily without locking the vessel calendar.
*   Once accepted and paid, the Proposal undergoes a state transition, spawning an active Booking.

## Examples

### Good Practice
*   Store temporary custom quotes inside a Proposal document until the guest completes deposit payment.
*   Link multiple proposed vessels to a single proposal so that guests can compare options.

### Bad Practice
*   Never lock a vessel calendar slot for a Proposal before receiving a deposit payment transaction.
*   Do not mix Booking transaction details inside a Proposal entity.
